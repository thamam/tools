# Markdown CodeLens Runner

A VSCode extension that adds **Copy** and **Run in Terminal** inline actions above every fenced code block in Markdown files.

```
$(copy) Copy   $(terminal) Run in Terminal (python)
```python
import os
print(os.getcwd())
` ``
```

## What it does

- Scans open `.md` files for fenced code blocks (backtick and tilde fences, any length ≥ 3)
- Shows two clickable CodeLens actions on the opening fence line:
  - **Copy** — copies the code block content to the clipboard
  - **Run in Terminal** — sends the code to a terminal named `Markdown Code Runner`, reusing it if already open

## Installation (new machine)

### Prerequisites

- [Node.js](https://nodejs.org/) (v18+)
- VSCode

### One-time setup

```bash
cd markdown-codelens-runner
npm install
npm run package
code --install-extension markdown-codelens-runner-*.vsix
```

Reload VSCode (`Cmd+Shift+P` → `Developer: Reload Window`) and open any `.md` file.

### Updating after code changes

```bash
npm run package
code --install-extension markdown-codelens-runner-*.vsix
```

VSCode will replace the existing version automatically.

## Development (F5 workflow)

Open the extension folder as its own VSCode workspace:

```bash
code markdown-codelens-runner/
```

Press **F5** — this compiles and launches a temporary Extension Development Host window with the extension loaded. Any `.md` file opened there will show the CodeLens actions.

Changes to the TypeScript source require recompiling (`npm run compile` or the watch task started by F5).

## How it works

**Parsing** (`markdownCodeLens.ts`): The document is scanned line-by-line. An opening fence is a line matching `` /^( {0,3})(`{3,}|~{3,})/ `` with an optional language identifier. The matching closing fence uses the same character and equal-or-greater length. Code content is everything between the two fences.

**CodeLens**: Two `vscode.CodeLens` objects are emitted per block, positioned at the opening fence line. Commands and their arguments (the code string) are bound at parse time — no deferred resolution needed.

**Terminal reuse**: `vscode.window.terminals` is searched for an existing terminal named `Markdown Code Runner` with `exitStatus === undefined`. If found it is reused; otherwise a new one is created.
